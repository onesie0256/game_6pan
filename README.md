## 注意
・masterブランチで作業しない!<br>
・コミットする時は「make clean」を実行して,objファイルを消す!<br>
・自分の進捗がある程度出来たら自分のブランチをmasterにマージする<br>
    →**このときエラーなくコンパイル出来,ちゃんとプログラムがコアダンプ無しにを動く状態にしなければならない!**

## makefileの使い方
そのまま「make」と打つと,サーバーの実行ファイルとクライアントの実行ファイルが作成します.  
しかし,クライアントのコード開発している人がサーバーのエラーのせいで,クライアントの実行ファイルが作成されないと激萎えすると思うので,
「make game_client」でクライアント,「make game_server」でサーバーのみ個別でコンパイルできます.
